# ï¿½ API Reference: Data Models

Dance Loop Gen uses **Pydantic** models to enforce strict structures for AI generation and internal data passing.

## 1. Loop Planning Models

### `Scene`
Defines a specific segment of the video motion.
- `scene_number` (int): Sequence index.
- `action_description` (str): Movement description.
- `audio_prompt` (str): Sound design guidance.
- `start_pose_description` (str): Description of the starting pose.
- `end_pose_description` (str): Description of the ending pose (loops to next scene).

### `ProjectConfig`
Main configuration for the generation run.
- `core_concept` (str): Central action (e.g., "Salsa dancing").
- `leader_description` (str): Appearance of the leader.
- `follower_description` (str): Appearance of the follower.
- `skip_music` (bool): If True, bypasses music prompt generation.
- `scene_variety` (int): 0-10 scale for background variation.

### `VideoPlan`
The master plan generated by the Director.
- `title` (str): Internal project title.
- `description` (str): Creative summary.
- `scenes` (List[Scene]): List of visual segments.

---

## 2. SEO & Metadata Models

### `MetadataOption`
A single SEO package.
- `title` (str): Recommended title.
- `description` (str): SEO-optimized description.
- `tags` (List[str]): Relevant hashtags.
- `emotional_hook` (str): The strategy used (e.g., "Nostalgic").

### `MetadataAlternatives`
The bundle of 3 options.
- `option_1`, `option_2`, `option_3` (MetadataOption).
- `recommended` (int): Index of the best choice.
- `reasoning` (str): AI's justification for the recommendation.

---

## 3. Batch Models

### `CSVRow`
Represents a single task in a batch CSV.
- `Created` (bool): Status flag.
- `Style` (str): Visual style.
- `Music` (str): Music genre.
- `Description` (str): Visual concept.
- `Scene_Variety` (Optional[int]): Local override for variety level.
